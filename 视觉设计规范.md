# Savor (余香) 视觉设计规范 (Design Guide)

> **设计基调**：Style B3 (Linear / Bold Minimal)
> **应用类型**：Desktop Application (Windows / macOS / Linux)

本规范旨在确保开发过程中 UI/UX 的一致性，打造专业、高效、沉浸的桌面端写作工具体验。核心理念是 **"Invisible Interface" (隐形界面)** —— 极致收敛的界面框架，让内容成为绝对主角。

---

## 1. 桌面端特性适配 (Desktop Native)

与 Web 页面不同，Savor 作为原生桌面应用，需要遵循以下交互与布局原则：

### 1.1 窗口结构 (Window Frame)
- **无边框窗口 (Frameless)**: 隐藏操作系统默认标题栏。
- **自定义拖拽区**: 顶部 `38px` 高度为可拖拽区域 (Traffic Lights / Window Controls 嵌入其中)。
- **布局模型**: 采用 **"Panes" (窗格)** 布局，而非 **"Pages" (页面)** 布局。
  - **侧边栏 (Sidebar)**: 固定位置，不随内容滚动。
  - **内容区 (Content)**: 独立滚动容器，拥有自己的滚动条。
  - **状态栏 (Status Bar)**: (可选) 底部固定，展示全局状态。

### 1.2 信息密度 (Information Density)
- **紧凑优先**: 相比 Web，桌面应用应提供更高的信息密度。
- **基准尺寸**:
  - 正文: `13px` / `14px` (Web 常为 16px)
  - 辅助文字: `11px` / `12px`
  - 列表行高: `28px` - `32px` (列表项)
  - 按钮高度: `28px` (Small), `32px` (Medium)

### 1.3 交互质感
- **Hover 响应**: 极速响应 (<50ms)，使用背景色微变（如 `#F7F7F7`），避免大幅度位移。
- **右键菜单 (Context Menu)**: 核心操作（如删除、重命名、导出）均需支持右键触发。
- **快捷键优先**: 界面上应在 Tooltip 中提示快捷键（如 `Cmd+S`, `Ctrl+N`）。

## 2. 视觉风格定义 (Visual Style)

### 2.1 色彩系统 (Color System)

基于 "Linear" 风格的黑白灰体系，克制使用彩色。

| 变量名           | 亮色模式 (Light) | 暗色模式 (Dark) | 用途                |
| :--------------- | :--------------- | :-------------- | :------------------ |
| `bg-app`         | `#FFFFFF`        | `#111111`       | 应用背景            |
| `bg-sidebar`     | `#FBFBFB`        | `#161616`       | 侧边栏/面板背景     |
| `bg-hover`       | `#F5F5F5`        | `#222222`       | 悬停/选中态背景     |
| `border-soft`    | `#EFEFEF`        | `#2A2A2A`       | 次要分割线          |
| `border-hard`    | `#E5E5E5`        | `#333333`       | 容器边框            |
| `text-primary`   | `#111111`        | `#EDEDED`       | 标题/正文           |
| `text-secondary` | `#707070`        | `#999999`       | 次要信息/标签       |
| `text-tertiary`  | `#A0A0A0`        | `#555555`       | 占位符/禁用态       |
| `accent`         | `#5E6AD2`        | `#6E7BF2`       | 品牌色 (Indigoblue) |

**原则**: 95% 的界面由黑白灰构成，`accent` 仅用于：
- 主按钮 (Primary Button)
- 选中状态的图标/文字
- 进度条/光标
- 极其重要的状态指示

### 2.2 排版 (Typography)

使用 **Inter** (Variable Font) 实现精准的字重控制。

- **Font Family**: `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif`
- **代码字体**: `'JetBrains Mono', 'Fira Code', monospace` (用于 Skill 规则编辑)

| 样式                | Size | Weight         | Line Height | Letter Spacing |
| :------------------ | :--- | :------------- | :---------- | :------------- |
| **H1** (Page Title) | 18px | 600 (SemiBold) | 1.2         | -0.02em        |
| **H2** (Section)    | 14px | 600 (SemiBold) | 1.4         | 0              |
| **H3** (Group)      | 12px | 600 (SemiBold) | 1.4         | 0              |
| **Body**            | 13px | 400 (Regular)  | 1.5         | 0              |
| **UI Interaction**  | 13px | 500 (Medium)   | 1.0         | 0              |
| **Caption/Label**   | 11px | 500 (Medium)   | 1.2         | 0.02em         |

### 2.3 图形与线条 (Graphics & Lines)

- **极细边框**: 使用 `1px` 物理像素边框。在高分屏上确保线条锐利。
- **直角与微圆角**:
  - 窗口/一级容器: 无圆角或极小圆角 (`0px` 或 `4px`)。
  - 按钮/输入框: `6px` 圆角。
  - Tag/Badge: `4px` 圆角。
- **无阴影 (Flat)**: 默认不使用阴影。仅在模态弹窗 (Modal) 或浮层 (Popover) 使用高扩散阴影 (`box-shadow: 0 8px 30px rgba(0,0,0,0.12)`) 以体现层级。

## 3. 核心组件规范 (Component Specs)

### 3.1 侧边栏 (Sidebar)
- 宽度: `240px` (可拖拽调整)
- 背景: `bg-sidebar`
- 选中项: 左侧无高亮条，整体背景变色及文字加粗。
- 图标: 16x16px，线性 (Stroke=1.5px)。

### 3.2 列表 (List Items)
桌面端核心交互载体。
- 高度: `32px` (Compact) / `36px` (Comfortable)
- 悬停: 全宽背景色变化。
- 选中: 背景变色 (`bg-hover`)。
- 操作: 鼠标悬停时右侧浮现“更多操作(...)”图标。

### 3.3 按钮 (Buttons)
- **Primary**: 黑色背景白字 (Dark Mode: 白底黑字)。无边框。
- **Secondary**: 透明背景，灰色文字，Hover 时变深色。
- **Outline**: 1px 边框 (`border-hard`)，透明背景。

### 3.4 编辑器区域 (Editor Canvas)
- **极简模式**: 隐藏所有顶部工具栏，仅保留类似 Typora 的写作区。
- **Markdown 渲染**:
  - 标题: 黑色，加粗。
  - 正文: 深灰 (`#333`)，行高 `1.6`。
  - 引用: 左侧 2px 竖线 (`border-hard`)，灰色文字。
  - 代码块: 浅灰背景 (`#F7F7F7`)，圆角 `4px`。

## 4. 关键界面状态 (Key Screens)

### 4.1 空状态 (Empty States)
不使用插画，仅使用：
- 一个线性图标 (48px, text-tertiary)
- 一句简短说明 (13px, text-secondary)
- 一个操作按钮 (如果适用)
保持极其“冷淡”的高级感。

### 4.2 加载状态 (Loading)
不使用全屏 Loading 转圈。
- 使用 **Skeleton Screen (骨架屏)**。
- 或在顶部显示极细的 **进度条 (Progress Bar)** (`height: 2px`, `accent` color)。

---

## 5. 开发实现建议 (Tips for Implementation)

1. **CSS Variables**: 必须严格使用定义的 CSS 变量，以便未来一键切换 Dark Mode。
2. **System Font Stack**: 不要动态加载网络字体文件（避免 FOUT 和额外的加载时间），直接使用系统字体栈，结合 `font-weight` 调整达到类似 Inter 的效果（除非打包时内嵌 Inter 字体的 WOFF2）。
3. **Cursor**: 确保可点击区域 cursor 为 `default` (菜单/列表) 或 `pointer` (按钮/链接)，桌面应用中大量菜单项使用 `default` 指针有时更符合原生体验（参考 macOS），但 Web 技术栈通常习惯 `pointer`，建议遵循 Web 习惯但保持克制。
